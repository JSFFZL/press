# 1. setup

  • vue3中的一个配置项，值是函数。
  • setup 是所有Composition API (组合API) “表演的舞台”。
  • 组件中所用到的：数据、方法等等，均要配置在setup 中。
  • setup 函数的两种返回值：

1、若返回一个对象，则对象中的属性、方法在模板中均可直接使用。（重点）

2、若返回一个渲染函数：则可以自定义渲染内容。（了解）

# 注意

1、尽量不要与vue2 配置混合用

2、 setup中不能访问到vue2配置 （data/ methos / computed）

3、setup不是一个async函数，因为返回值不再是return的对象，而是promise，模板看不到return对象中的属性。

4、在beforeCreate 之前执行一次，this是undefined。

5、setup 的参数：

  • props：值是对象，包含：组件外部传递过来，且组件内部声明接收了的属性。
  • context：上下文对象
  • attrs：值是对象，包含：组件外部传递过来，但没有在props配置中声明的属性，相当于this.$attrs。
  • slots：收到的插槽内容，相当于this.$slots。
  • emit：分发自定义事件的函数，相当于this.$emit。

# 2.ref

作用：定义一个响应的数据

语法：const xx = ref(value);

创建一个包含响应式数据的引用对象。

js中操作数据：xx.value

模板中读取数据不需要 .value ,直接<div></div>

备注：

  • 接收的数据可以是：基本类型，也可以是对象类型。
  • 基本类型的数据：响应式依然是靠object.defineProperty() get 和 set 完成的。
  • 对象类型的数据：内部使用了vue3的---reactive 函数
<script>
    // vue3 所用到的函数都必须先引入（ref,reactive）
	import {ref，reactive} from "vue";
    export default {
		setup() {
            /**
			 * 数据
			 */
			let num = ref(10);
			let obj = {
				"name":"张三",
				"age":18
			};
			let db = [
				{"label":"jack","value":"123"},
				{"label":"lucy","value":"456"}
			];
         /**
		 * 方法
		 */
		function min() {
			num.value = num.value - 1; //注意，修改数据的值需要.value
		};
       // 返回数据与方法
        return {
			num,
			obj,
			db,
            min
		}
    }
}
</script>

# 3.reactive

reactive：定义一个对象类型响应式数据 (一般使用的多)

原理是：proxy 实现，通过代理对象操作源对象内部数据都是响应式的。

reactive 定义对象类型（或数组），是把数据放到一个对象中去。

常用写法：

setup(){
    let data = reactive({
        person:{},
        student:{}
    })
    return {
        data
    }
}

# ref 和 reactive区别：

1、定义数据对比：

  • ref 基本类型数据。
  • reactive 对象（数组）类型数据。

2、原理对比：

  • ref 通过object.defineProperty()的get和set 实现响应式（数据劫持）
  • reactive 通过 proxy 实现响应式（数据劫持），并通过Reflect 操作源对象内部的数据。

3、使用对比：

  • ref 定义的数据：操作数据需要.value , 读取数据时模板中读取不需要.value。
  • reactive定义数据：操作数据与读取数据：均不需要.value。

reflect （反射）

# 4.生命周期钩子函数

你可以通过在生命周期钩子前面加上 “on” 来访问组件的生命周期钩子。

下表包含如何在 setup () (opens new window) 内部调用生命周期钩子：

选项式 API Hook inside setup
beforeCreate Not needed*
created Not needed*
beforeMount onBeforeMount
mounted onMounted
beforeUpdate onBeforeUpdate
updated onUpdated
beforeUnmount onBeforeUnmount
unmounted onUnmounted
errorCaptured onErrorCaptured
renderTracked onRenderTracked
renderTriggered onRenderTriggered
activated onActivated
deactivated onDeactivated

TIP

因为 setup 是围绕 beforeCreate 和 created 生命周期钩子运行的，所以不需要显式地定义它们。换句话说，在这些钩子中编写的任何代码都应该直

接在 setup 函数中编写。

# 生命周期内请求数据

这些函数接受一个回调函数，当钩子被组件调用时将会被执行:

// MyBook.vue
import {onMounted} from "vue";
export default {
  setup() {
    // mounted
    onMounted(() => {
      console.log('Component is mounted!')
    })
  }
}

# 5.计算属性与监视

1、computed 函数

  • 与vue2配置功能一致

  • 写法：

    import {computed} from 'vue'
setup(props) {
    let person = reactive({
        names:"", //名字
        age:"", // 年龄
        message:""  // 信息：名字+年龄
    })
    // 计算属性-简写形式（只读）
    person.message = computed(()=>{
        return person.names + "+" + person.age
    })
    // 计算属性-完整写法（修改）
    person.message = computed({
        get(){ //读取message 是调用 get 返回names和age的值
            return person.names + "+" + person.age
        },
        set(value){ // 修改实时的值
            let newArr = value.split("+");
            person.names = newArr[0];
            person.age = newArr[1];
        }
    })
    return {
        person
    }
},

# 6.配置文件，配置别名

在配置文件内：（根目录位置和src同级）

vue.config.js （重启生效）

module.exports = {
    configureWebpack: {
        resolve: {
            alias:{
                'assets':'@/assets'， //配置路径，前面是别名，后面是路径 （绝对位置）
                'compontens':'@/conpontens', //可以设置多个别名
                '':''
            }
        }
    },
    publicPath:'./'
}
